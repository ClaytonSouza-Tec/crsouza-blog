Azure Migrate: Passo a Passo para Migrar Servidores para o Azure

Um guia prático para inventariar, avaliar, replicar e executar cutover de workloads on-premises com o Azure Migrate.

Quando usar o Azure Migrate?

Azure Migrate é a plataforma central da Microsoft para descoberta, avaliação e migração de servidores, bancos de dados e aplicações para o Azure. Ele reduz riscos porque permite validar sizing, custos e dependências antes de qualquer desligamento em produção.

Na prática, o fluxo recomendado é: descobrir o ambiente atual, avaliar impacto e custo, replicar as máquinas e só então fazer o cutover.

Pré-requisitos antes de começar

  • Subscription Azure ativa com permissões para criar recursos
  • Resource Group dedicado para a migração
  • Conectividade entre ambiente local e Azure (VPN/ExpressRoute ou internet com portas liberadas)
  • Conta com acesso administrativo para instalar o appliance de descoberta
  • Planejamento de janela de mudança para o cutover

Passo 1: Criar o projeto no Azure Migrate

  1. No portal Azure, busque por Azure Migrate
  2. Clique em Create project
  3. Selecione subscription, resource group, geografia e nome do projeto
  4. Finalize em Review + Create

Esse projeto será o hub onde você acompanha descoberta, avaliação e status das ondas de migração.

Passo 2: Configurar descoberta com o appliance

O appliance é uma VM que coleta inventário e metadados de performance no seu datacenter. Para cenários VMware/Hyper-V, o procedimento é:

  1. No projeto, escolha Discovery and assessment
  2. Baixe o appliance correspondente ao seu ambiente
  3. Implante a VM local e acesse o painel de configuração
  4. Registre o appliance com a chave gerada no Azure Migrate
  5. Informe credenciais de descoberta para mapear as VMs

Após alguns minutos, as máquinas descobertas começam a aparecer no inventário do projeto.

Passo 3: Criar assessments (sizing e custo)

Com o inventário pronto, o próximo passo é montar uma avaliação para simular a execução no Azure.

  1. Abra Assessments e clique em Create assessment
  2. Defina o tipo de ambiente (VMware, Hyper-V ou físico)
  3. Escolha critério de sizing: por performance (recomendado) ou on-premises
  4. Selecione região alvo, tipo de storage e política de reserva
  5. Revise readiness, custo estimado e dependências

Use esse relatório para priorizar quick wins e separar workloads que precisam de remediação prévia.

Passo 4: Preparar replicação para Azure VM

Depois da avaliação, inicie a replicação das máquinas para o Azure com o serviço de migração de servidores.

  1. No projeto, acesse Azure Migrate: Server Migration
  2. Selecione Replicate e escolha as VMs de origem
  3. Defina subscription, VNet, subnet e resource group de destino
  4. Escolha tamanho da VM no Azure e opções de disco
  5. Inicie a replicação contínua

Esse processo mantém as alterações sincronizadas até o momento do corte final.

Passo 5: Test migration (obrigatório)

Antes do corte real, execute um teste para validar rede, aplicação e acesso sem impactar produção.

  1. Em cada VM replicada, clique em Test migrate
  2. Use uma VNet de teste isolada
  3. Valide inicialização, serviços, portas e autenticação
  4. Documente ajustes necessários e execute novo teste se preciso

Somente após esse checklist a migração deve avançar para cutover.

Passo 6: Cutover e pós-migração

  1. Agende a janela de mudança com os times envolvidos
  2. Interrompa serviços na origem conforme o runbook
  3. Execute Migrate para promover a réplica no Azure
  4. Valide aplicação, DNS, monitoramento e backups
  5. Se tudo estiver estável, faça o decommission da origem

Após o cutover, habilite monitoramento com Azure Monitor e proteja os dados com Azure Backup para fechar o ciclo operacional.

Boas práticas para reduzir risco

  • Migre por ondas pequenas, começando por sistemas menos críticos
  • Padronize naming, tags e políticas de governança antes do cutover
  • Use grupos de aplicação para preservar dependências entre VMs
  • Mantenha runbook com plano de rollback por workload
  • Revise custo real pós-migração para aplicar right-sizing

Conclusão

Azure Migrate organiza a jornada de migração em etapas previsíveis, com visibilidade técnica e financeira desde o início. Com descoberta correta, assessment bem calibrado e testes de migração, você reduz indisponibilidade e acelera a transição para o Azure com muito mais segurança.
